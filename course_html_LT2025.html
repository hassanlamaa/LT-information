Foundations of Client/Server Model

🔍 What is Client/Server?

A network model where each computer is either a client or a server. Clients rely on servers for resources like files, devices, and processing power.

  • Clients: PCs/workstations running user applications
  • Servers: Powerful computers storing/managing public files and applications

🔄 Communication Pattern

Clients and servers exchange messages using a request-response pattern through Inter-Process Communication (IPC).

They communicate using common protocols and servers use scheduling systems to handle multiple client requests.

⚖️ Pros & Cons

Advantages:

  • Centralized management
  • Scalability
  • Enhanced data sharing
  • Improved security

Disadvantages:

  • Server cost
  • Complex configuration
  • Single point of failure
  • Traffic congestion

Server Types & System Architectures

🖥️ Server Types

  • File Server: Shares files across network
  • Database Server: Provides database services
  • Transaction Server: Handles OLTP operations
  • Groupware Server: Enables collaboration
  • Web Server: Handles HTTP protocol
  • Web Application Server: Supports dynamic content

⚖️ Fat Client vs Fat Server

Fat Client: More functions on client side, knows data organization

Fat Server: More functions on server side, easier to manage and deploy

🏗️ N-Tier Architectures

  • 2-Tier: Client + Database Server
  • 3-Tier: Client + Application Server + Database Server
  • N-Tier: Extended architecture with specialized components

2-Tier vs 3-Tier Architecture Comparison

Feature 2-Tier 3-Tier
System Administration More logic on client to manage Centrally managed on server
Performance Poor (many SQL statements over network) Good (only service requests/responses)
Scalability Poor/Low Excellent (distribute load across servers)
Application Reuse Poor (application on client) Excellent (reuse services and objects)
Ease of Development High Complex

Core Communication Technologies

🔌 Sockets & Ports

Sockets provide communication mechanism between computers using IP addresses and ports.

Well-known ports:

  • HTTP: 80
  • HTTPS: 443
  • FTP: 21
  • SMTP: 25
  • DNS: 53

🔄 TCP vs UDP

TCP: Connection-oriented, reliable, ordered delivery

UDP: Connectionless, faster, no delivery guarantees

Used for voice, video, DNS, DHCP

☕ Java Networking

Java provides comprehensive networking support:

  • InetAddress class for IP address representation
  • Socket and ServerSocket for TCP communication
  • DatagramSocket for UDP communication

TCP vs UDP Detailed Comparison

Feature TCP UDP
Connection Connection-oriented Connectionless
Reliability High (error checking, retransmission) Low (no guarantees)
Speed Slower Faster
Header Size 20-60 bytes 8 bytes
Usage HTTP, HTTPS, FTP, SMTP DNS, DHCP, VoIP, Video Streaming 
// Java TCP Client Example
import java.io.*;
import java.net.*;

public class TCPClient {
    public static void main(String args[]) throws Exception {
        String sentence;
        String modifiedSentence;
        BufferedReader inFromUser = new BufferedReader(
            new InputStreamReader(System.in));
        
        Socket clientSocket = new Socket("localhost", 6789);
        
        DataOutputStream outToServer = new DataOutputStream(
            clientSocket.getOutputStream());
        
        BufferedReader inFromServer = new BufferedReader(
            new InputStreamReader(clientSocket.getInputStream()));
        
        sentence = inFromUser.readLine();
        outToServer.writeBytes(sentence + '\n');
        modifiedSentence = inFromServer.readLine();
        System.out.println(modifiedSentence);
        clientSocket.close();
    }
}

Application Protocols (Case Studies)

🌐 HTTP Protocol

Application-level protocol for distributed, collaborative, hypermedia information systems.

Key Features:

  • Connectionless
  • Media independent
  • Stateless

📁 FTP Protocol

Protocol for transferring files between computers, typically from your computer to a web server.

Uses port 20 for data and port 21 for control.

🔒 HTTP Methods

  • GET: Retrieve information
  • POST: Send data to server
  • PUT: Replace resource
  • DELETE: Remove resource
  • HEAD: Get headers only

Middleware & Advanced Distribution

🔗 What is Middleware?

Connectivity software that enables multiple processes to interact across a network.

Essential for migrating mainframe applications to client/server and providing communication across heterogeneous platforms.

📊 Middleware Types

  • RPC - Remote Procedure Calls
  • MOM - Message-Oriented Middleware
  • Publish/Subscribe
  • ORB - Object Request Broker
  • SQL-oriented Data Access

🗃️ Database Middleware

  • ODBC - Open Database Connectivity
  • OLE-DB - Microsoft enhancement
  • JDBC - Java Database Connectivity

JDBC Architecture

Java Application → JDBC API → JDBC Driver Manager → JDBC Driver → Database

// JDBC Connection Example
import java.sql.*;

public class JDBCExample {
    static final String JDBC_DRIVER = "com.mysql.jdbc.Driver";
    static final String DB_URL = "jdbc:mysql://localhost/EMP";
    static final String USER = "username";
    static final String PASS = "password";
    
    public static void main(String[] args) {
        Connection conn = null;
        Statement stmt = null;
        try {
            // Register JDBC driver
            Class.forName(JDBC_DRIVER);
            
            // Open a connection
            conn = DriverManager.getConnection(DB_URL, USER, PASS);
            
            // Execute a query
            stmt = conn.createStatement();
            String sql = "SELECT id, first, last, age FROM Employees";
            ResultSet rs = stmt.executeQuery(sql);
            
            // Extract data from result set
            while(rs.next()) {
                int id = rs.getInt("id");
                int age = rs.getInt("age");
                String first = rs.getString("first");
                String last = rs.getString("last");
                
                System.out.print("ID: " + id);
                System.out.print(", Age: " + age);
                System.out.print(", First: " + first);
                System.out.println(", Last: " + last);
            }
            rs.close();
            stmt.close();
            conn.close();
        } catch(SQLException se) {
            se.printStackTrace();
        } catch(Exception e) {
            e.printStackTrace();
        }
    }
}

Remote Invocation (RPC & RMI)

📞 RPC - Remote Procedure Call

Mechanism to execute subroutines on other hosts without including socket code in programs.

Hides network code into stub procedures.

☕ RMI - Remote Method Invocation

Java mechanism allowing objects in one JVM to invoke objects running on another JVM.

Used to build distributed applications.

🌐 CORBA

Common Object Request Broker Architecture - platform and language-independent framework for building distributed systems.

RPC Mechanism

Client → Client Stub → Network → Server Stub → Server Procedure

Then reverse for response

// RMI Remote Interface Example
import java.rmi.Remote;
import java.rmi.RemoteException;

public interface Calculator extends Remote {
    public long add(long a, long b) throws RemoteException;
    public long sub(long a, long b) throws RemoteException;
    public long mul(long a, long b) throws RemoteException;
    public long div(long a, long b) throws RemoteException;
}
