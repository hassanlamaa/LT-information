HTTP vs HTTPS — The core difference

HTTP

HyperText Transfer Protocol — plain text, no encryption. Data sent in clear text, vulnerable to eavesdropping (man-in-the-middle). Default port: 80.

insecure

HTTPS

HTTP + SSL/TLS encryption. Authenticates server and encrypts all data, ensuring confidentiality & integrity. Default port: 443. Essential for modern web.

secure

Key takeaway: HTTPS protects login credentials, payments, and sensitive data. Always prefer HTTPS in production.

HTTP Request Methods (verbs)

HTTP methods indicate the desired action to perform on a resource. Below are the most important ones:

GET
read / retrieve data
safe + idempotent
POST
create new resource
not idempotent
PUT
full update / replace
idempotent
PATCH
partial update
not always idempotent
DELETE
remove resource
idempotent

⚡ Important concepts: Safe methods (GET) don't alter server state. Idempotent methods (PUT, DELETE, GET) have same effect after multiple identical requests. POST is neither safe nor idempotent.

Example request lines:
GET /users/42 HTTP/1.1
POST /products HTTP/1.1 (with body: { "name": "Laptop" })
DELETE /posts/7 HTTP/1.1
