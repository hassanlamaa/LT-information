Understanding Client-Server Architecture

Client-server architecture is a distributed application structure that partitions tasks or workloads between service providers (servers) and service requesters (clients).

Correct! In 1-tier architecture, all components exist in a single system without separation.
2 A 2-tier architecture is also known as a client-server architecture.
True
False
Correct! The 2-tier architecture is the classic client-server model where the client handles presentation and business logic, while the server manages data storage.
3 In a 3-tier architecture, the business logic is typically located on the client side.
True
False
Correct! In 3-tier architecture, business logic is separated into its own tier (application tier), not located on the client side.
4 N-tier architecture allows for better scalability than 2-tier architecture.
True
False
Correct! N-tier architecture provides better scalability as each tier can be scaled independently based on demand.
5 The main advantage of 1-tier architecture is its high scalability for large user bases.
True
False
Correct! 1-tier architecture has very poor scalability as all components are combined in a single system.
6 In client-server architecture, the server always initiates the communication with clients.
True
False
Correct! In client-server architecture, clients typically initiate communication by requesting services from servers.
7 A web browser is an example of a client in client-server architecture.
True
False
Correct! A web browser acts as a client that requests web pages from web servers.
8 Peer-to-peer architecture is the same as client-server architecture.
True
False
Correct! Peer-to-peer architecture is different from client-server architecture as each node can act as both client and server.
9 Thick clients contain most of the application logic and require minimal server interaction.
True
False
Correct! Thick clients (fat clients) process most data locally and only communicate with the server for data storage/retrieval.
10 Microservices architecture is an example of N-tier architecture.
True
False
Correct! Microservices architecture extends the N-tier concept by further decomposing application tiers into independently deployable services.
