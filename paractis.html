🔸 Descriptions

[1] Routes requests and acts as a switch between objects.
[2] Allows servers to receive requests without pre‑compiled skeletons (runtime binding).
[3] Stores available object interfaces (types, methods) for dynamic lookup.
[4] Mediates ORB and servants, handles activation & reference generation.

IDL Coding Challenge

Write an IDL interface for a TemperatureSensor that:

  • Has a readonly attribute location (string).
  • Has an operation getTemp() returning a float.
  • May raise a user exception SensorError (with no extra data).
Hint 
module Sensors {
  exception SensorError {};
  interface TemperatureSensor {
    readonly attribute string location;
    float getTemp() raises (SensorError);
  };
};
