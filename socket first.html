The InetAddress Class

The InetAddress class represents an Internet Protocol (IP) address in Java. It's used to perform hostname resolution and IP address manipulation.

Creating InetAddress Instances

InetAddress has no public constructor, so you must obtain instances via static factory methods:

Example: Creating InetAddress Objects 

import java.net.InetAddress;
import java.net.UnknownHostException;

public class InetAddressDemo {
    public static void main(String[] args) {
        try {
            // Get local host address
            InetAddress localHost = InetAddress.getLocalHost();
            System.out.println("Local Host: " + localHost);
            
            // Get address by hostname
            InetAddress google = InetAddress.getByName("google.com");
            System.out.println("Google: " + google);
            
            // Get all addresses for a hostname
            InetAddress[] allGoogle = InetAddress.getAllByName("www.google.com");
            System.out.println("All Google addresses:");
            for (InetAddress addr : allGoogle) {
                System.out.println("  - " + addr);
            }
            
            // Get address from IP string
            InetAddress fromIP = InetAddress.getByName("8.8.8.8");
            System.out.println("From IP: " + fromIP);
            System.out.println("Host Name: " + fromIP.getHostName());
            
        } catch (UnknownHostException e) {
            System.out.println("Error: Could not resolve hostname");
            e.printStackTrace();
        }
    }
}

Key Methods of InetAddress

  • getLocalHost() - Returns the InetAddress of the local machine
  • getByName(String host) - Returns InetAddress for a hostname or IP string
  • getAllByName(String host) - Returns all InetAddresses for a hostname
  • getHostName() - Returns the hostname for this IP address
  • getHostAddress() - Returns the IP address string
  • isReachable(int timeout) - Tests if address is reachable

Socket Programming Basics

Sockets provide the communication mechanism between two computers over a network. A socket is one endpoint of a two-way communication link.

Client-Server Communication Model

Client: Creates a socket and attempts to connect to a server

Server: Listens for connections and creates a socket for each client

Socket Address: IP Address + Port Number (e.g., 192.168.1.10:8080)

TCP vs UDP Sockets

TCP (Transmission Control Protocol)

Connection-oriented

Reliable - guarantees delivery

In-order delivery of packets

Stream-based communication

Examples: HTTP, HTTPS, FTP, Email

UDP (User Datagram Protocol)

Connectionless

Unreliable - no delivery guarantees

No ordering of packets

Datagram-based communication

Examples: DNS, VoIP, Online games

Socket Addressing

Every socket is identified by a combination of:

  • IP Address: Identifies the host (e.g., 192.168.1.10)
  • Port Number: Identifies the application/service (e.g., 80 for HTTP)
Port Classification 

// Well-known ports (0-1023): Reserved for system services
// HTTP: 80, HTTPS: 443, FTP: 21, SSH: 22, SMTP: 25

// Registered ports (1024-49151): Assigned by IANA for specific services

// Dynamic/Private ports (49152-65535): For temporary use by clients
// Also called ephemeral ports

TCP Socket Programming in Java

Java provides two key classes for TCP socket programming:

  • java.net.Socket - For client-side communication
  • java.net.ServerSocket - For server-side listening and acceptance

TCP Connection Lifecycle

1. Server creates a ServerSocket and binds to a port

2. Server calls accept() and waits for client connections

3. Client creates a Socket and connects to server

4. Connection established - data exchange begins

5. Communication via InputStream and OutputStream

6. Connection closed by either party

Example: Simple TCP Server 

import java.net.*;
import java.io.*;

public class SimpleTCPServer {
    public static void main(String[] args) throws IOException {
        // Create server socket listening on port 12345
        ServerSocket serverSocket = new ServerSocket(12345);
        System.out.println("Server started. Waiting for client...");
        
        // Wait for client connection
        Socket clientSocket = serverSocket.accept();
        System.out.println("Client connected: " + 
                          clientSocket.getInetAddress());
        
        // Create input and output streams
        PrintWriter out = new PrintWriter(
            clientSocket.getOutputStream(), true);
        BufferedReader in = new BufferedReader(
            new InputStreamReader(clientSocket.getInputStream()));
        
        // Read client message
        String message = in.readLine();
        System.out.println("Received: " + message);
        
        // Send response
        out.println("Echo: " + message);
        
        // Close connections
        in.close();
        out.close();
        clientSocket.close();
        serverSocket.close();
    }
}
Example: Simple TCP Client 

import java.net.*;
import java.io.*;

public class SimpleTCPClient {
    public static void main(String[] args) throws IOException {
        // Connect to server on localhost, port 12345
        Socket socket = new Socket("localhost", 12345);
        
        // Create input and output streams
        PrintWriter out = new PrintWriter(
            socket.getOutputStream(), true);
        BufferedReader in = new BufferedReader(
            new InputStreamReader(socket.getInputStream()));
        
        // Send message to server
        out.println("Hello Server!");
        
        // Read server response
        String response = in.readLine();
        System.out.println("Server response: " + response);
        
        // Close connections
        in.close();
        out.close();
        socket.close();
    }
}

Exception Handling in Socket Programming

Always handle these common exceptions:

  • UnknownHostException - Host cannot be resolved
  • ConnectException - Connection refused
  • SocketTimeoutException - Connection timeout
  • IOException - General I/O errors
