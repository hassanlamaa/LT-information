📡 4.9.3 RMI‑IIOP & Java IDL

IIOP (Internet Inter‑ORB Protocol) is CORBA’s standard protocol (TCP/IP) enabling cross‑language distributed communication. CORBA (Common Object Request Broker Architecture) – OMG standard for platform‑neutral distributed objects.

Java IDL = Java’s CORBA/IIOP implementation. The idlj compiler generates stubs and skeletons from IDL files, allowing Java objects to interact with any CORBA‑compliant system.

// Example IDL (hello.idl)
module HelloApp { interface Hello { string sayHello(); }; }; // idlj -fall hello.idl → generates client stubs + server skeletons

RMI‑IIOP: combines RMI simplicity with CORBA interoperability. By adhering to restrictions (no pass‑by‑reference, etc.), RMI remote objects can use IIOP and communicate with CORBA clients in any language (C++, Python, etc.).

🏛️ 4.9.4 OMG CORBA – Introduction

CORBA is a technical standard for an ORB (Object Request Broker) – an object‑oriented evolution of RPC. It allows applications on different computers, OS, CPU types, and languages (C++, Java, Python, COBOL) to interoperate.

“Making software work together” – middleware for heterogeneous environments.

  • Distributed middleware – network transparency.
  • Language‑independent interface definitions via IDL.
  • ORB handles location, activation, communication.
// CORBA invocation steps at runtime:
1. Server object starts, registers with ORB.
2. Client binds (finds) server object reference via ORB.
3. Client invokes method on object reference.
4. ORB delivers request to server object.
5. Server returns result through ORB.

⚙️ CORBA Architecture – Core Components


Interface Repository
Dynamic info about object interfaces.

DII
Dynamic Invocation Interface – construct requests at runtime.

Client Stubs (IDL)
Precompiled static invocations.

ORB Interface
APIs for local ORB services.

Object Adapter
Mediates ORB & servants (BOA/OA).

Server Skeletons
Static IDL skeletons for server methods.

DSI (Dynamic Skeleton)
Runtime binding for servers without precompiled stubs.

Implementation Repository
Stores server activation details.

ORB Core is the switch – routes requests between objects. Object Adapter (e.g., POA – Portable Object Adapter) handles activation, reference generation, request dispatching.

📜 Interface Definition Language (IDL) & Example

IDL is declarative – defines object interfaces (operations, attributes, exceptions). Supports in / out / inout parameter directions, multiple inheritance, and user exceptions.

// Example IDL file: Bank.idl
module Bank {
exception InsufficientFunds {};
interface Account {
readonly attribute string owner;
long balance();
void withdraw(in long amount) raises (InsufficientFunds);
void deposit(in long amount);
};
};

Compiled with idlj -fall Bank.idl → stubs, skeletons, helper classes. System exceptions (30+ predefined) are thrown by ORB; user exceptions defined in IDL.

✅ Advantages of CORBA & Role as Middleware

  • Language, OS, hardware independence.
  • Location transparency – objects can move without client change.
  • Built‑on standard IDL – strict interface contracts.
  • Interoperability via IIOP.
  • Reuse of legacy systems by wrapping them as CORBA objects.

Middleware (CORBA, RMI, MQ, .NET) connects applications. CORBA is distributed middleware – excels at cross‑platform, cross‑language communication.
