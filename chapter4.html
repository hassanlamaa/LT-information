4.1 Introduction to RPC

Remote Procedure Call allows a program to execute subroutines on another host without manually coding sockets. Network details are hidden inside stub procedures.

  • Analogy with local function call: push parameters, call, return.
  • Why RPC? Enables distributed computing with familiar procedure-call semantics.
  • Characteristics: different process/machine, no shared address space, parameters passed by value (no pointers).
  • Features: simple syntax, well-defined interface, efficient, hides network complexity.

4.2 RPC Model & Mechanism

Client stub marshals arguments → sends message → server dispatcher → server stub unmarshals → calls server procedure → result marshaled back.

Data representation: XDR (eXternal Data Representation) ensures machine independence. Marshalling = assemble into external format; unmarshalling = rebuild local format.

Asynchronous RPC: client continues after acknowledgment, server processes later. One-way RPCs (fire-and-forget).

4.3 Identification: Program, Version, Procedure Numbers

Each RPC procedure is uniquely identified by:
Program number (group of related procedures)
Version number (evolve protocols smoothly)
Procedure number (specific function).
Sun defined ranges: 0x00000000 – 0x1fffffff (Sun), 0x20000000 – 0x3fffffff (user-defined).

4.4 Dynamic Port Mapping & Port Mapper

RPC programs cannot use fixed ports because program numbers exceed 16-bit port range. Solution: Port mapper (port 111) maintains a dynamic mapping from (program number, version) to protocol port. Servers register; clients query.

Server registers: (RPC program number, protocol port)
Client lookup: portMapper.Lookup(programNumber) → port

4.5 RPC Call Semantics

Exactly once (hard), At most once (error if unsure), At least once (repeat until success, for idempotent procedures). Idempotent = repeated execution same effect (e.g., read account balance).

4.6 RPC Authentication

NULL (no auth), UNIX style (hostname+UID/GID, weak), DES (encrypted, strong), SHORT (handle after initial auth).

4.8 RPC Language & rpcgen Compiler

- Client stub, server skeleton, XDR routines, header file.
- Simplifies distributed app development dramatically.

4.9 Practical: Sun Java RMI (Remote Method Invocation)

RMI allows Java objects to invoke methods on remote JVMs. Architecture: Stub (client proxy), Skeleton (server proxy), Remote Reference Layer, Transport. RMI Registry (rmiregistry, default port 1099).

// Remote interface public interface Calculator extends Remote { long add(long a, long b) throws RemoteException; } // Implementation & Server export, Client lookup via Naming.lookup()

Example: Calculator with add/sub/mul/div, also MySQL database RMI example (retrieve student_data table).
Steps: define remote interface → implement → create server with bind() → client lookup().
Security: policy file grants permissions (socket, file).
