The InetAddress Class

📍 Java's representation of an IP address - both IPv4 and IPv6

Key Characteristics:

  • No public constructor - use static factory methods
  • Immutable and thread-safe
  • Handles both hostname resolution and IP address representation
Creating InetAddress Instances
// Get local machine address
InetAddress localHost = InetAddress.getLocalHost();

// Resolve hostname to IP
InetAddress google = InetAddress.getByName("google.com");

// Get all addresses for a host
InetAddress[] allAddresses = 
    InetAddress.getAllByName("www.google.com");

// Create from IP string
InetAddress fromIP = InetAddress.getByName("8.8.8.8");

InetAddress: Key Methods

🏠
getLocalHost()

Returns address of local machine

🔤
getByName(String)

Resolves hostname to IP address

📚
getAllByName(String)

Returns all IPs for a hostname

Practical Example
import java.net.*;

public class InetAddressDemo {
    public static void main(String[] args) {
        try {
            InetAddress local = InetAddress.getLocalHost();
            System.out.println("Local Host: " + local);
            
            System.out.println("Host Name: " + local.getHostName());
            System.out.println("IP Address: " + local.getHostAddress());
            
            // Test reachability (may require admin on some systems)
            boolean reachable = local.isReachable(5000);
            System.out.println("Is reachable: " + reachable);
            
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Socket Programming Basics

🔌 Sockets are endpoints for two-way communication between machines

Socket = IP Address + Port Number

Example: 192.168.1.10:8080 (IP:Port)

Client Socket
  • Initiates connection to server
  • Created with remote IP and port
  • java.net.Socket class
  • Example: Web browser connecting to a website
Server Socket
  • Listens for incoming connections
  • Binds to specific port
  • java.net.ServerSocket class
  • Example: Web server waiting for requests

TCP vs UDP: Choosing the Right Protocol

TCP
Transmission Control Protocol
  • Connection-oriented - requires handshake
  • Reliable - guarantees delivery
  • In-order delivery
  • Flow control and congestion control
  • Higher overhead, slower
  • Use cases: Web, email, file transfer
UDP
User Datagram Protocol
  • Connectionless - no handshake
  • Unreliable - no delivery guarantees
  • No ordering guarantees
  • No flow control
  • Lower overhead, faster
  • Use cases: DNS, VoIP, gaming, streaming

Simple Rule:

Need reliability? Use TCP. Need speed? Use UDP. Need both? Implement reliability on top of UDP.

TCP Socket Programming in Java

🔧 Two key classes: Socket (client) and ServerSocket (server)

TCP Connection Lifecycle
// Server Side
ServerSocket serverSocket = new ServerSocket(12345);
Socket clientSocket = serverSocket.accept(); // Waits here

// Client Side  
Socket socket = new Socket("localhost", 12345);

// Both sides can now communicate via:
// socket.getInputStream()
// socket.getOutputStream()

// Always close sockets when done!
socket.close();
serverSocket.close();

Important Notes:

  • ServerSocket.accept() blocks until a client connects
  • Each client connection creates a new Socket instance
  • Always use try-with-resources or finally block to close sockets
  • Handle exceptions: UnknownHostException, IOException, SocketException

Complete TCP Example: Echo Server

Echo Server Implementation
import java.net.*;
import java.io.*;

public class EchoServer {
    public static void main(String[] args) throws IOException {
        ServerSocket serverSocket = new ServerSocket(12345);
        System.out.println("Echo Server started on port 12345");
        
        while (true) {
            Socket clientSocket = serverSocket.accept();
            System.out.println("Client connected: " + 
                             clientSocket.getInetAddress());
            
            // Handle client in new thread for multiple clients
            new Thread(() -> handleClient(clientSocket)).start();
        }
    }
    
    private static void handleClient(Socket socket) {
        try (BufferedReader in = new BufferedReader(
                new InputStreamReader(socket.getInputStream()));
             PrintWriter out = new PrintWriter(
                socket.getOutputStream(), true)) {
            
            String inputLine;
            while ((inputLine = in.readLine()) != null) {
                System.out.println("Received: " + inputLine);
                out.println("Echo: " + inputLine);
            }
        } catch (IOException e) {
            System.err.println("Error handling client: " + e.getMessage());
        }
    }
}

Key Features Shown:

• Multi-threaded server (handles multiple clients) • Proper resource management • Clean separation of concerns

Network Exception Handling

⚠️ Network operations can fail in many ways - always handle exceptions!

🔍
UnknownHostException

Hostname cannot be resolved to IP address

🚫
ConnectException

Connection refused by remote host

⏰
SocketTimeoutException

Connection or read timeout expired

Best Practice: Robust Error Handling
try {
    InetAddress address = InetAddress.getByName("example.com");
    Socket socket = new Socket(address, 80);
    socket.setSoTimeout(5000); // 5 second timeout
    
    // ... communication code ...
    
} catch (UnknownHostException e) {
    System.err.println("Cannot resolve hostname: " + e.getMessage());
    // Suggest checking network/DNS
} catch (SocketTimeoutException e) {
    System.err.println("Connection timed out");
    // Retry logic or notify user
} catch (IOException e) {
    System.err.println("Network error: " + e.getMessage());
    // General network failure
} finally {
    // Always clean up resources
}
